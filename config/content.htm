報告1

Tetris 線上遊戲專案報告2

我發現當我將老師所寫的那一串程式中間直接加入已寫的Tetris程式網頁就出現Tetris這個遊戲的介面，操作結果為下。

配置過程:

先在自己的動態網頁中Edit All加入一個目錄，後點選該目錄的Edit ，進入後在點選Source code插入下列html文字檔中的程式，將其複製到Source code中，在中間加入Tetris程式碼，按下save，在按viewpage回到目錄中，檢查Tetris是否出現在該頁面上，若出現並正常執行則代表成功，後將動態網頁push成靜態網頁看看在動態網頁中能否看到Tetris。

此為程式html文字檔:

<!-- 導入 Brython 程式庫 -->

<script src="/static/brython.js"=導入之路徑></script>

<script src="/static/brython_stdlib.js"=導入之路徑></script>

<!-- 啟動 Brython -->

<script>

window.onload=function(){brython();}=此方法用于在網頁加載完畢後立即執行的操作</script>

<script type="text/python3">=在 HTML 文件的 <script> 標記內運行 python 代碼

#print("可執行 Brython")=用print來執行Brython


<<需顯示之遊戲程式碼>>


</script>

def clear_bd(ev):

    bd.clear()
# Brython 3.3.4 內建的 container 名稱為  'container' 且 turtle 輸出為 svg 必須使用 div 訂定 id
Ace = ace.Editor(editor_id="kw_editor", console_id="kw_console", container_id="kw__container", storage_id="kw_py_src" )
# 從 gist 取出程式碼後, 放入 editor 作為 default 程式
def run():
    # 利用 get 取下 src 變數值
    try:
        url = doc.query["src"]
    except:
        url = "插入個人gists中的程式文字檔網址"
    prog = open(url).read()

    # 將程式載入編輯區
    Ace.editor.setValue(prog)
    Ace.editor.scrollToRow(0)
    Ace.editor.gotoLine(0)
    # 直接執行程式
    #ns = {'__name__':'__main__'}
    #exec(prog, ns)
    # 按下 run 按鈕
    Ace.run()

# 執行程式, 顯示輸出結果與清除輸出結果及對應按鈕綁定
doc['kw_run'].bind('click', Ace.run)
doc['kw_show_console'].bind('click', Ace.show_console)
doc['kw_clear_console'].bind('click', Ace.clear_console)
doc['clear_bd'].bind('click', clear_bd)
# 呼叫函式執行
run()
// ]]></script>

onedrive

因其只有5GB，當備份太多時，則電腦內的磁碟會無發運行。
